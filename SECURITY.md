# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 3.0.x   | ✅ Active  |
| 2.x     | ❌ Legacy  |

## Reporting Security Issues

If you find a security vulnerability **in PathProbe itself** (not in targets you're testing):

1. **Do NOT** open a public issue
2. Email: [your-email@example.com]
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact

## Responsible Use

PathProbe is a security testing tool. Users must:
- Have **written authorization** before testing any system
- Follow their organization's security testing policies
- Comply with all applicable laws and regulations
- Not use this tool for unauthorized access or malicious purposes

## Scope

This security policy covers the PathProbe tool itself. Vulnerabilities found
using PathProbe in other applications should be reported to those applications'
respective security teams through their responsible disclosure processes.
